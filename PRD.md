# 产品需求文档（PRD）

**项目名称：** 多车驾驶模拟器系统 · 3D 可视化展示
**文档版本：** v1.1
**日期：** 2026-06-16

> **v1.1 变更说明：** 为便于数据维护与后续扩展，实现时由「单文件 HTML」调整为**前后端分离架构**——前端 Vue 2.7 + Vite + Three.js，后端 Node.js + Express 提供拓扑数据 REST API。后端不可用时前端自动回退到内置数据，仍保证可独立运行。详见第 6、7 章。

---

## 1. 产品概述

本项目是一个基于 Web 技术的 **三维驾驶仿真系统可视化展示页面**，用于直观呈现多类型驾驶模拟器与 Master-Agent 仿真网络的物理布局、设备架构及实时数据流向。通过沉浸式 3D 场景，帮助使用者理解整套仿真系统的硬件拓扑与通信关系。

---

## 2. 背景与目标

| 项目     | 说明                                                                                                                   |
| -------- | ---------------------------------------------------------------------------------------------------------------------- |
| 业务背景 | 驾驶模拟器实验室需要一套可视化工具，将多类型模拟器与服务器网络的连接关系以直观形式展示给参观者、研究人员及系统管理人员 |
| 核心目标 | 以 3D 场景还原真实仿真实验室的设备布局，实时展示 Master 与 Agent 服务器之间、Agent 与模拟器之间的数据同步流向          |
| 交付形式 | 单文件 HTML，无需后端，浏览器直接打开即可运行                                                                          |

## 4. 功能需求

### 4.1 场景内容

#### 4.1.1 模拟器

系统包含 3 类模拟器：

| 编号 | 类型         | 颜色标识         | 主要构成                                                     |
| ---- | ------------ | ---------------- | ------------------------------------------------------------ |
| 1    | 机动车模拟器 | 红色 `#cc4422` | 运动平台、驾驶舱（方向盘、油门踏板、刹车踏板、挡位）、投影屏 |
| 2    | 助动车模拟器 | 橙色 `#dd8800` | 运动平台、骑行座舱（车把）、投影屏                           |
| 3    | 自行车模拟器 | 绿色 `#44aa44` | 运动平台、自行车骨架（前后轮、车架、把手）、投影屏           |

**共用组件：**

- **运动平台（Motion Platform）：** 深色金属质感底板，带蓝色边缘发光描边
- **投影屏幕：** 带有道路场景纹理（透视公路 + 城市天际线），带蓝色发光外框

#### 4.1.2 服务器

| 模块   | 说明                         |
| ------ | ---------------------------- |
| Master | 核心节点，负责管理各个 Agent |
| Agent  | 分布式节点的代理节点         |

- 分布式架构采用 **主从模式**
- 每一台服务器内分别有一个Agent
- 第一台为主服务器，是Master + Agent
- 每个 Agent 连接一台模拟器

#### 4.1.3 连接线与数据粒子

| 连接类型               | 颜色             | 说明                               |
| ---------------------- | ---------------- | ---------------------------------- |
| Agent → 模拟器 数据流 | 青色 `#00ccff` | 二次贝塞尔弧线，低弧度（arcH=1.8） |

**数据粒子动画：**

- 青色粒子沿 Agent→模拟器曲线循环运动，速度 0.006/帧
- 连接线透明度呼吸式脉冲，周期约 3.5 秒

#### 4.1.4 场景环境

- **地面：** 深色金属地板 + 蓝灰色网格线（100×100 单位，50 格）
- **雾效：** 深蓝色背景雾，近 55 单位起始，远 130 单位消散
- **背景色：** 深夜蓝黑 `#040810`

## 5. 非功能需求

### 5.1 性能

| 指标     | 要求                                 |
| -------- | ------------------------------------ |
| 渲染帧率 | 主流设备 60 FPS                      |
| 像素密度 | 限制最大 DPR = 2，防止高分屏性能损耗 |
| 阴影质量 | PCFSoftShadowMap，2048×2048         |
| 粒子数量 | 6 个（轻量级）                       |

### 5.2 兼容性

| 环境   | 要求                                                                         |
| ------ | ---------------------------------------------------------------------------- |
| 浏览器 | 支持 ES Module Import Map 的现代浏览器（Chrome 89+、Edge 89+、Firefox 108+） |
| 设备   | 桌面端为主，支持触控设备旋转/缩放                                            |
| 网络   | Three.js 库通过 jsDelivr CDN 加载，需要网络连接                              |

### 5.3 部署

- 单文件 HTML，零依赖本地安装
- 直接双击或通过本地 HTTP 服务器打开

## 6. 系统实现架构（v1.1 新增）

实现采用前后端分离：

| 层   | 技术                          | 职责                                        |
| ---- | ----------------------------- | ------------------------------------------- |
| 前端 | Vue 2.7 + Vite 5 + Three.js   | 3D 场景渲染、交互、UI 面板                  |
| 后端 | Node.js 18+ + Express（ESM）  | 提供仿真拓扑数据 REST API                    |

**目录结构：**

```
DIL/
├── backend/    # Node.js + Express API（端口 3000）
└── frontend/   # Vue2 + Vite + Three.js（端口 5173）
```

**数据流：** 前端启动时请求 `GET /api/topology`；若后端不可用则回退到内置数据并提示。
拓扑数据唯一来源为 `backend/src/data/topology.js`，`frontend/src/services/fallbackData.js` 为其镜像。

**API 接口：** `/api/topology`、`/api/simulators(/:id)`、`/api/servers(/:id)`、`/api/connections`、`/api/health`。

详细架构见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)，协作规范见 [CLAUDE.md](CLAUDE.md)。
